# EDA 주요 발견 사항



## 데이터 개요

- **기간**: 2010-12-01 ~ 2011-12-09 (373일)

- **총 거래**: 541,909건

- **고유 고객**: 4,372명

- **고객당 평균 구매**: 93.1회

- **1회 구매 고객**: 79명 (재구매율 98.2%)



## 데이터 품질 이슈



### 1. 결측치

| 컬럼 | 결측 건수 | 비율 | 해석 |
|------|----------|------|------|
| CustomerID | 135,080건 | 24.9% | 비회원 구매로 추정 |
| Description | 1,454건 | 0.27% | 상품명 누락 (소량) |


**처리 방침:**

- DB 적재 시 그대로 보존

- 코호트/RFM 분석: CustomerID NULL 제외

- 전체 매출 분석: NULL 포함



### 2. 이상치



#### 음수 Quantity (10,624건)

- InvoiceNo가 'C'로 시작 → 취소/반품 주문

- 모든 'C' 인보이스가 음수 수량 (100% 일치)

- **처리**: SQL 뷰에서 `Quantity > 0` 필터링



#### 음수 UnitPrice (2건)

- 명백한 데이터 오류

- **처리**: 전처리에서 제거



### 3. 중복 데이터

- 완전 중복 행: 5,268건

- 동일 송장 내 동일 상품 복수 구매 가능성

- **처리**: 삭제하지 않고 분석 단위에 맞게 집계



### 4. 특수 코드



**InvoiceNo 패턴:**

- 'C'로 시작 (9,288건): 취소/반품 주문

- 'A'로 시작 (3건): 용도 불명

- 숫자만 (532,618건): 정상 주문



**StockCode 특수 코드:**

- POST (1,256건): 배송비

- DOT (710건): 할인

- M/m (572건): Manual (대소문자 중복)

- AMAZONFEE, BANK\_CHARGES 등: 수수료



**처리 방침:**

- StockCode 대문자 통일 검토

- 특수 코드는 분석 목적에 따라 포함/제외

## 전처리 방침



1. **날짜 타입 변환** - datetime으로 변환 (코호트 분석 필요)

2. **결측치 보존** - DB에 그대로 적재, 뷰에서 목적별 처리

3. **음수 가격 제거** - 2건의 명백한 오류 제거

4. **음수 수량 보존** - 반품 데이터이므로 보존, 뷰에서 필터링

5. **StockCode 대문자 통일** - 'M'/'m' 같은 중복 방지 (보류-선택사항)

6. **중복 데이터 보존** - 집계 시 처리

7. **특수 코드 처리** - 분석 목적에 따라 추후 결정
